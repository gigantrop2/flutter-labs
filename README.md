SmartCalc - Умный калькулятор
Краткое описание
SmartCalc — это мобильное приложение-калькулятор с дополнительной полезной функцией: история вычислений. Приложение простое, интуитивно понятное и решает повседневные задачи.

Основные функции (разбивка по этапам)
Этап 1: ЛР4 (Верстка)

Базовый калькулятор: Экран с цифровыми кнопками (0-9), основными операторами (+, -, *, /, =), кнопкой сброса (C) и десятичной точкой.

Экран истории: Простой список всех предыдущих вычислений.

Навигация: Нижнее или боковое меню для переключения между двумя основными экранами.

Этап 2: ЛР5 (Логика)

Логика вычислений: Корректная обработка нажатий кнопок, выполнение арифметических операций, соблюдение порядка действий.

Логика истории: Сохранение в память приложения каждой выполненной операции в формате "5 + 3 = 8" и отображение в списке.

Логика навигации: Реализация перехода между экранами.

Этап 3: ЛР6 (Данные)

Локальное хранение: Сохранение истории вычислений в shared_preferences или локальной БД (SQLite), чтобы история не сбрасывалась после перезапуска приложения.

Работа с API: Получение актуального времени с внешнего API для отметки времени вычислений.

Структура экранов и навигация

text

1. Экран калькулятора
• Верхняя часть: Дисплей для отображения вводимого выражения и результата.
• Основная часть: Сетка кнопок:
C, ⌫, %, ÷
7, 8, 9, ×
4, 5, 6, -
1, 2, 3, +
0, ., =
• Навигация: Нижняя панель для перехода на экран истории.

2. Экран истории

Заголовок: "История вычислений".

Основная часть: ListView с прокруткой, отображающий последние операции (самая свежая сверху). Каждый элемент показывает выражение, результат и время вычисления.

Действие: При тапе на запись из истории, выражение можно перенести в калькулятор для повторного использования или редактирования.

Кнопка очистки истории.

Чек-лист необходимого функционала для MVP

Экран: 2 экрана (Калькулятор, История).

Навигация: Реализована через BottomNavigationBar.

Интерактивность: Ввод цифр/операторов.

Список данных: История вычислений отображается в виде списка.

Хранение данных: История сохраняется локально (shared_preferences).

Работа с API: Запрос к API времени для отметки вычислений.

Стилизация: Единая цветовая схема.
